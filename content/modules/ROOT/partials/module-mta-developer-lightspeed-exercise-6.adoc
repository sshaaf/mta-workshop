=== Overview

In Part 1 (the inventory walkthrough in Exercise 5), the **Inventory Management** app was migrated from enterprise audit library v1 to v2 with Konveyor AI. The **Solution Server** learned that migration and stored the patterns, solutions, and success metrics. This part demonstrates how the solution server **speeds up** migration of the **EHR** application with **contextual hints** and **confidence** levels from those learned patterns.

=== Prerequisites

* Complete Exercise 5 (Part 1: **inventory** migration with the same `openjdk_custom_library` style profile and custom audit rules) or an equivalent run where the **Solution Server** has captured accepted solutions
* link:https://github.com/hhpatel14/ehr_viewer/tree/java8[EHR Application] (clone from `java8` branch in the steps below)
* Same **development environment** and tooling as in Exercise 5 (for example **VS Code** / **Dev Spaces**, **Java 21**, **Maven**, **Podman** for **Logstash** as documented)

image::mta-dls/exercise-6/2_3_solution.jpg[Logger solution,link=self,window=blank,title="Logger solution"]

The **Solution Server** can apply the same **StreamableAuditLogger** configuration that worked in Exercise 5.

[source,java]
----
AuditConfiguration config = new AuditConfiguration();
config.setStreamHost(System.getenv().getOrDefault("AUDIT_STREAM_HOST", "localhost"));
config.setStreamPort(Integer.parseInt(System.getenv().getOrDefault("AUDIT_STREAM_PORT", "5000")));
config.setStreamProtocol(System.getenv().getOrDefault("AUDIT_STREAM_PROTOCOL", "tcp"));
auditLogger = new StreamableAuditLogger(config);
----

* **Environment-based** settings for different deployments
* **Defaults** for local use (`localhost:5000`)
* **TCP** to **Logstash** or your central pipeline
* **StreamableAuditLogger** wiring consistent with the inventory app

Apply the change in the **EHR** service classes and save as directed by the **resolution** / **fix** flow.

=== Step 4: Build and run the migrated application

==== 4.1 Updated POM configuration

Update `pom.xml` with the needed **Spring Boot 3** / **Java 21** alignment (example **fragment** from the upstream walkthrough; align with the actual `ehr_viewer` repository as you import changes):

[source,xml]
----
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0
         http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.0.0</modelVersion>

    <parent>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-parent</artifactId>
        <version>3.2.0</version>
        <relativePath/>
    </parent>

    <groupId>com.example</groupId>
    <artifactId>ehr-viewer</artifactId>
    <version>2.0.0</version>
    <packaging>jar</packaging>

    <name>Electronic Health Record Viewer</name>
    <description>Spring Boot 3 application for EHR management with enhanced audit logging</description>
----

[NOTE]
====
The upstream example uses a shortened `+pom+` snippet; your real project POM on the `java21` (or target) branch may differ. Use the EHR application’s own module coordinates and parent versions when merging.
